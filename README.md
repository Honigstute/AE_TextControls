# Text Controls

After Effects subtitle/text helper with SRT import, local transcription, OpenAI
transcription, and text decomposition/preset tools.

## Download

Download the public package from this repository:

```text
TextControls-v1.0.24-public.zip
```

Unzip it first. The package contains the After Effects ZXP, the macOS
dependency installer, one README, and legal notices.

The Windows dependency manager is built by GitHub Actions from the
`installer-windows/` folder. After the workflow finishes, download the artifact
named `TextControlsDependencies-win-x64` and place
`TextControlsDependencies-win-x64.exe` into the public package before uploading
the final Windows-ready ZIP.

## Installation

Text Controls is a ZXP extension and can be installed using a ZXP installer.

---

### Step 1 — Download a ZXP Installer

Download and install a ZXP installer.

Recommended:
[aescripts + aeplugins ZXP Installer](https://aescripts.com/learn/post/zxp-installer)

---

### Step 2 — Install the Plugin

- Open the ZXP Installer
- Drag & drop the `TextControls-v1.0.24.zxp` file from the unzipped package into the installer
- Follow the installation process

---

### Step 3 — Launch After Effects

Open After Effects and find the panel under:

```text
Window -> Extensions -> Text Controls
```

---

## Transcription Setup

Text Controls supports two transcription paths:

- `OpenAI Whisper API`
- `Local`

---

### OpenAI Whisper API

This is the easiest setup.

You need:

- an OpenAI API key

Recommended setup:

- `Engine`: `OpenAI Whisper API`
- `OpenAI Render`: `Slow Render (compatibility)`

With that setup, you do not need:

- Python
- local Whisper
- ffmpeg

OpenAI speech-to-text docs:
[Speech to text](https://platform.openai.com/docs/guides/speech-to-text)

OpenAI transcription API reference:
[Audio transcriptions API](https://platform.openai.com/docs/api-reference/audio/createTranscription)

OpenAI model reference:
[whisper-1](https://developers.openai.com/api/docs/models/whisper-1)

Note:

- `Fast Render (ffmpeg needed)` requires `ffmpeg`
- file uploads are limited by OpenAI
- OpenAI currently supports common upload formats such as `mp3`, `mp4`, `mpeg`, `mpga`, `m4a`, `wav`, and `webm` according to the current speech-to-text guide

---

### Local on macOS

macOS now has an installer-managed local runtime.

You need:

- the ZXP panel
- `TextControlsInstaller.app`

The macOS installer is included inside the public ZIP as:

- `TextControlsInstaller-macos.zip`

The installer app manages:

- `~/Library/Application Support/Text Controls Local/bin/tc-whisper-helper`
- `~/Library/Application Support/Text Controls Local/bin/ffmpeg`
- `~/Library/Application Support/Text Controls Local/models/ggml-base.bin`
- `install-manifest.json`

Important:

- no Python is required for this macOS local path
- no Homebrew is required for this macOS local path
- FFmpeg is downloaded by the installer and kept inside the Text Controls local runtime
- selected-file local transcription uses AVFoundation and supports common macOS-readable files such as `wav`, `mp3`, `m4a`, `mp4`, `aiff`, and `caf`
- selected-comp local transcription works because the panel renders comp audio to a temporary file first

### Local on Windows

Windows now uses an installer-managed local runtime, matching the macOS
direction but packaged as a portable dependency manager.

You need:

- the ZXP panel
- `TextControlsDependencies-win-x64.exe`

The Windows dependency manager is generated by GitHub Actions:

- [installer-windows/README.md](installer-windows/README.md)
- [.github/workflows/windows-dependencies.yml](.github/workflows/windows-dependencies.yml)

The dependency manager manages:

- `%LOCALAPPDATA%\Text Controls Local\bin\tc-whisper-helper.exe`
- `%LOCALAPPDATA%\Text Controls Local\bin\whisper-cli.exe`
- `%LOCALAPPDATA%\Text Controls Local\bin\ffmpeg.exe`
- `%LOCALAPPDATA%\Text Controls Local\models\ggml-base.bin`
- `install-manifest.json`

Important:

- no Python is required for this Windows local path
- no `openai-whisper` is required for this Windows local path
- no admin rights are required because the runtime is installed per user
- FFmpeg and the default Whisper model are downloaded by the dependency manager and checksum-verified
- unsigned Windows builds can show SmartScreen / Unknown Publisher warnings until a Windows code-signing certificate and reputation are available

## Windows / macOS Notes

If you install the panel with a ZXP installer, the extension install itself is simple on both systems.

If you install manually, Adobe CEP documents these extension folders:

- Windows per-user: `C:\Users\<USERNAME>\AppData\Roaming\Adobe\CEP\extensions`
- macOS per-user: `~/Library/Application Support/Adobe/CEP/extensions`

Adobe CEP extension folder reference:
[Adobe CEP HTML Extension Cookbook](https://github.com/Adobe-CEP/CEP-Resources/blob/master/CEP_12.x/Documentation/CEP%2012%20HTML%20Extension%20Cookbook.md)

Practical recommendation:

- on Windows, use `TextControlsDependencies-win-x64.exe` for the installer-managed local runtime
- on macOS, use `TextControlsInstaller.app` for the installer-managed local runtime

---

## Notes

- ZXP plugins require a separate installer because Adobe no longer supports direct double-click installation.
- Make sure After Effects is closed during installation.
- If the panel does not appear, restart After Effects.
- For the easiest public-facing setup, use `OpenAI Whisper API` with `Slow Render (compatibility)`.
- For fully offline transcription on macOS, use the installer-managed local runtime.
- For fully offline transcription on Windows, use the installer-managed local runtime.
- When transcribing a selected comp, the plugin creates a temporary audio export automatically.
- When transcribing a media file, the plugin can use the selected file directly.
